1. What is the data-type (e.g. string or integer) of the information that comes in? What happens if the user inputs a number -- does it stay an integer? Use `typeof` and `console.log()` to help with this

This from webdevelopernotes.com: It's important to remember that the value returned by prompt() and subsequently stored in a variable will always be a string data type. This is fine if you are dealing with text, but might present problems to the code if you plan to receive numeric data through prompt(). Javascript provides two functions to convert this string value to a numeric data type; parseInt() and parseFloat(). The parseInt() converts a string to an integer value while parseFloat() parses the string converting it to a floating point number.

4. If you completely changed a question you were asking the user, would it make sense to totally rename the variable? What if you changed the order of your questions?

That's tricky. I would think that if I changed the question enough that a reasonable person would provide different inputs from the previous asking of the question, I would rename the variable. It might be a case where we would not want to mingle this data with data captured previously. We might even be changing this intentionally, to see if we get "better" answers. Changing the order of the questions can have a similar effect. In a political questionnaire, for example, the first few questions might establish a mindset or steer the user to an opinion expressed as a response to a later question. Changing the order of the questions can change the responses, so yes, I would probably change the variable name.
